# 用 Vibe Coding 做一台桌面 AI 终端 — M5Tab5 教程大纲

> 这是什么:一台 M5Stack Tab5(ESP32-P4 + 5″ 触摸屏)被改造成「桌面智能终端」——
> 能控制全屋智能家居、跟 AI 语音对话、看股票行情、收邮件提醒、玩 LED 灯阵……
>
> **关键点:整个项目几乎不用手写代码。** 你只要会"说人话"——把想要的功能描述给
> AI 编程助手(比如 Claude Code),它来写代码、编译、烧录;你在真机上看效果、提意见,
> 它再改。这套"边聊边做"的方式就叫 **Vibe Coding**。
>
> 本大纲面向**完全没嵌入式经验的小白**,告诉你如何靠对话一步步把这台终端做出来。

## 二、什么是 Vibe Coding(核心理念)

传统写代码:你得懂 C/C++、Python、编译工具链、硬件寄存器……门槛极高。

Vibe Coding:**你负责"想要什么"和"对不对",AI 负责"怎么实现"。**

- 你说:「帮我加一个自选股 App,能显示行情,涨的红色跌的绿色」
- AI:读懂现有代码 → 写新代码 → 编译 → 帮你烧到设备
- 你:在屏幕上看,「颜色反了,中国习惯涨红跌绿」
- AI:改一行 → 重新烧 → 你再看

**你不需要看懂每一行代码,但你需要:**
1. 把需求说清楚(越具体越好)
2. 在真机上认真测,把"哪里不对"准确反馈
3. 对"要不要烧录/要不要花钱/要不要删东西"这类决定拍板

## 四、Vibe Coding 的循环(整个项目的节奏)

```
   ┌─────────────────────────────────────────────┐
   │ 1. 你用人话描述想要的功能 / 要改的地方          │
   │        ↓                                      │
   │ 2. AI 读代码、写代码                           │
   │        ↓                                      │
   │ 3. AI 编译(build)—— 有错它自己修              │
   │        ↓                                      │
   │ 4. 【你拍板】要不要烧录到设备?                 │
   │        ↓                                      │
   │ 5. AI 烧录(flash)                            │
   │        ↓                                      │
   │ 6. 你在真机上看效果、动手测                     │
   │        ↓                                      │
   │ 7. 你反馈"哪里不对" → 回到 1                    │
   └─────────────────────────────────────────────┘
```

> 经验:**烧录前一定让 AI 停下来问你**(烧录会动真机,值得确认)。这是本项目反复强调的一条规矩。

## 五、怎么"聊"出每个功能(把功能当作对话示例)

下面每一节 = 一个你可以照着说的需求。难度从易到难。

### 5.1 起步:先有个能开机的桌面
- 你说:「帮我搭一个多 App 的启动器主屏,每个图标点进去是一个独立功能」
- 得到:`AppHome` 主屏 + Mooncake 框架管理每个 App 的生命周期 + 状态栏(WiFi/天气/邮件)

### 5.2 智能家居面板
- 你说:「接我的 Home Assistant,做一个能开关灯、空调、电视的触屏面板,洗衣机要显示剩余时间」
- 得到:`app_ha`,直连 HA 的几百个设备,分灯光/家电/卡片/天气几个 tab

### 5.3 小智语音助手
- 你说:「把开源的小智语音助手嵌进来,能唤醒词唤醒、语音对话」
- 得到:`app_xiaozhi`(内嵌 xiaozhi-esp32,本地唤醒词)

### 5.4 Hermes AI 助手(本项目的灵魂)
- 你说:「做一个打字跟 AI 对话的 App,要有记忆,能帮我安排定时任务,做危险操作前要在屏幕上问我同不同意」
- 得到:`Hermes` App —— 触屏聊天 + 流式回复 + **记忆**(跨多轮)+ **定时任务** + **触屏权限审批**
- 背后:Mac 上一个"桥"程序,把屏幕的聊天转成标准的 Agent 协议(ACP)去驱动 Hermes 智能体
- 💡 小白也能体会的设计智慧:**不重新造 UI**——直接把原来的"Claude 对话 App"改名换后端成 Hermes,键盘/语音/历史全复用

### 5.5 自选股
- 你说:「做个自选股 App,7 列行情,点开看分析,LED 灯阵滚动播报,涨红跌绿」
- 得到:`app_stocks` + 后端转发行情 API + LED 滚动 ticker
- 体会:一句"颜色反了"就能让 AI 改对——**中国 A 股涨红跌绿**

### 5.6 邮件提醒 / LED 灯阵 / LoRa / 小工具
- 「有未读邮件时让 LED 滚动显示 NEW EMAIL」→ 后台 `AppEmailLed`
- 「5 块灯板拼成跑马灯,能显示我打的字」→ `app_unit_puzzle`
- 「加个 LoRa 聊天,和另一台设备远距离收发」→ `app_lora_chat`
- 「加个计算器 / 汇率 / 单位换算」→ 工具页

### 5.7 给 AI 助手加"能力"(进阶)
- 你说:「让小智能语音控制云鲸扫地机」
- 得到:后端给 AI 加了一个"扫地机工具"(开始/暂停/回充/查电量…),小智一句话就能调

## 六、AI 也会犯错:你怎么帮它发现(真实避坑)

Vibe Coding 不是"许愿就成",**你在真机上的反馈是关键**。本项目真实踩过的坑:

| 现象(你看到的) | 真因(AI 帮你查出来的) | 怎么修 |
|---|---|---|
| LED 文字"一闪一闪"看着累 | 清屏函数会先把灯全灭再刷,等于每帧黑一下 | 改成"只清缓冲、末尾刷一次" |
| LED 绿色看着发蓝 | 屏幕用的祖母绿带蓝光,灯上偏色 | LED 改用纯红/纯绿三原色 |
| 小智说关电视显示成功但电视没反应 | 那个开关是"音箱模式"且**逻辑反的** | 把开关方向反过来 |
| 退出 App 后 LED 还在放股票 | 框架退出时不触发收尾回调 | 用"当前是不是前台"来判断并收尾 |
| 让 AI 干活总说"权限不足" | AI 在无人值守模式下没法弹框确认 | 接通屏幕审批卡,你点同意 |
| 想让设备朗读回复但没声音 | 设备通用音频通道被禁用(和屏幕抢 I2C) | 暂时搁置,留作专项 |

> 心法:**把"哪里不对"描述得越具体,AI 定位越快。** "没声音"不如"点了喇叭键、桥日志显示音频已发出但设备不响"。

## 七、构建与烧录(可以全程让 AI 代劳)

你完全可以让 AI 跑这些命令;贴在这里是为了让你心里有数:

```bash
# 准备环境
source ~/.local/bin/idf_env.sh

# 编译(idf.py 偶尔卡,用 ninja 更稳)
cd ~/Projects/M5Tab5/M5Tab5-UserDemo/platforms/tab5
ninja -C build

# 烧录(USB 口名字偶尔会变)
PORT=$(ls /dev/cu.usbmodem* | head -1)
python -m esptool --chip esp32p4 -p "$PORT" -b 460800 \
  --before default_reset --after hard_reset write_flash \
  --flash_mode dio --flash_size 16MB --flash_freq 80m \
  0x2000 build/bootloader/bootloader.bin \
  0x8000 build/partition_table/partition-table.bin \
  0x10000 build/m5stack_tab5.bin \
  0xd74000 build/srmodels/srmodels.bin
```

## 八、给小白的几条心法

1. **一次只改一件事**:做完一个小功能就烧录看效果,别一口气堆十个需求。
2. **真机为准**:模拟器/日志能帮忙,但最终对不对要在设备上亲眼看、亲手点。
3. **学会拍板**:烧录、花钱、删文件、改公开内容——这些 AI 会停下来问你,你要会决定。
4. **取舍也是设计**:本项目加过 Bing 壁纸屏保,后来为省空间又删了;能加也要敢减。
5. **让 AI 记住你的偏好**:比如"用中文回复""烧录前先问我",说一次它就会一直照做。
